# OKE Chip Compiler for Carnage Heart Portable

- カルネージハート EXA 版は[こちら](https://github.com/yoshinrt/okecc/tree/che)
- カルネージハート Zeus/Zeus II 版は[こちら](https://github.com/yoshinrt/okecc/tree/zeus)

OKE Chip Compiler (okecc) は，カルネージハート ポータブルの OKE ソフトを C 言語に似た[プログラミング言語](https://github.com/yoshinrt/okecc/blob/chp/sample.cpp)で記述し，チップ配置を自動で行うソフトウェアです．
チップ配置ではなく，okecc とプログラミング言語 (テキスト) で OKE ソフトを設計することは，以下のような利点があります．

- 「チップを並べる」行為から解放されます
	- チップ移動や矢印修正，チップ配置整形，NOP 圧縮の手間がありません．
	- 例えば既存のソフトに 1chip 挿入する場合，okecc では，1行追加するだけで済みます．
	- 他の OKE ソフトをコピペするだけで流用できます．
- プログラミング言語で書かれたソフトは，保守・管理が容易です
	- コメントで，ソフトの意図を明確に記述することができます．
	- `If` `Else` `End` などの制御構文等，複雑な条件分岐も簡潔に記述できます．

また，okecc は以下のような特徴を持っています．

- メソッドチェーンを使用した簡潔な命令記述
	- パラメータ数が多い命令も簡潔に記述できます．例: `fire(1, 5).dist(200).span(90).wait`
- 無制限のサブルーチン数 (インラインサブルーチン)
	- インラインサブルーチンは呼び出し位置にインライン展開されます．
	- 実際の SUB1/2 に割り当てを変更することも容易です．
- C++ との高い親和性
	- okecc は C++ 文法を拡張しているため，C++ 文法もそのまま使用できます．
	- 例: `#define` で変数にわかりやすい変数名をつける，`for` 文による繰り返しチップ配置，等
- svg ファイルによるグラフィカルなチップ配置表示
	- 配置結果を svg に出力し，ゲームに読み込む前に確認できます．

# 使用方法

Windows (Microsoft Visual Studio または Cygwin)，Linux 環境で動作します．

## Windows 環境 (Microsoft Visual Studio)

1. 本ソフトの動作には Microsoft Visual Studio が必要ですのでインストールしてください．Microsoft Visual Studio Community 2022 で動作確認しています．
1. 本 git リポジトリをクローンします．
1. powershell プロンプトを立ち上げ，git リポジトリをクローンしたディレクトリに移動します．
1. 以下のコマンドを実行すると，サンプルの OKE ソフトがコンパイルされます．
	```

	```

## 実行時間について

チップ配置は力任せで探索しているため，実行時間はチップ数と PC の CPU パワーと運に依存します．

参考までに，Ryzen5 5600G での実行時間は以下のとおりです．(配置探索時間のみ)

- 45チップ (sample.cpp): 3秒以下
- 198チップ (test_all_chip.cpp): およそ 20-30秒

## 交差矢印について

本コンパイラが生成するチップ配置は，チップ配置成功率を上げるため，交差矢印を禁止していません．

# OKE ソフトの記述方法

## メインルーチンの記述

OKE ソフトは C 言語に似た言語で記述します．以下に，最小限の OKE ソフト記述を示します．

```
#include <okecc.h>

void chip_main(){
	// ここに命令を記述
}
```

`chip_main()` が，メインの 15x15 のエリアを意味し，ここに配置するプログラムを記述します．`chip_main()` は必ず必要です．

命令は，OKE ソフトチップに相当する命令を記述します．使用できる命令は[命令一覧](#命令一覧)を参照してください．

## 命令記述の基本

パラメータがない最も基本的な命令は，命令名とセミコロン `;` を記述します．
```
fight; // 格闘攻撃
```

パラメータを持つ命令は，命令名の後にカッコ `()` を記述し，カッコ内にパラメータを記述します．
```
option(1); // オプション 1 を動作させる
```

エリア指定パラメータを持つ攻撃命令など，特にパラメータ数が多い命令は，パラメータ指定を省略することができます．省略した場合は，その命令のデフォルト値が使用されます．省略せずに指定する場合は，命令の直後に ピリオド `.` を続けてオプション名を指定します．

```
// 武装 1を 5発発射．以下のパラメータはデフォルト値が使用されます
//   エリア指定: 極座標系 距離 800m, 方位 0°, 幅 360°
//   射撃モード: 通常射撃
//   実行モード: 通過
fire(1, 5);

// すべてのパラメータをオプションで指定した例
fire(1, 5).dist(800).dir(0).span(360).snipe.wait;
```

どのような命令・パラメータ・オプションが使用できるかは，[命令一覧](#命令一覧)を参照してください．

## サブルーチンの記述

以下に，サブルーチンの記述例を示します．

```
#include <okecc.h>

void sub_routine(){
	startSub(1);
	
	// ここに SUB1 ソフトを記述
}

void chip_main(){
	...
	// SUB1 の呼び出し
	sub_routine();
	...
}
```

サブルーチンとして記述する関数 (ここでは `sub_routine()`) を記述し，その先頭で `startSub(n)` を記述します．`n` には 1 または 2 を指定し，それぞれ SUB1, SUB2 に配置されます．また関数名は自由に定義できます．

サブルーチン名直後の `()` は省略できないので注意してください．

### インラインサブルーチンの記述

インラインサブルーチンは，生成されたチップは SUB1/SUB2 に配置されるのではなく，そのサブルーチン呼び出し位置に展開されます．
チップ数が許す限り，インラインサブルーチンは何個でも定義・呼び出しが可能です．

以下に，インラインサブルーチンの記述例を示します．

```
#include <okecc.h>

void inline_sub_routine(){
	startInlineSub;
	
	// ソフトを記述
}

void chip_main(){
	...
	// インライサブルーチンのチップをここに展開
	inline_sub_routine();
	...
}
```
`startSub(n)` の代わりに `startInlineSub` を記述することで，インラインサブルーチンとなります．

## Return 命令

`Return` 命令は，そのサブルーチンを抜けて呼び出し元に戻ります．

- MAIN/SUB1/SUB2 では，チップ配置エリアの外枠に向かうチップが配置されます．
- インラインサブルーチンでは，呼び出し位置に戻ります．

## 条件分岐の記述

条件分岐には，`If` `ElseIf` `Else` `End` を使用して記述します．以下に記述例を示します．

```
If(numAmmo(1) >= 100)
	// 武装 1残弾が 100 発以上の時の処理
	...
ElseIf(isTargetStopped)
	// 武装 1残弾が 100 未満 かつ ターゲットが静止中の処理
	...
Else
	// 上記条件のいずれにも当てはまらないときの処理
	...
End
```

`If(...)` および `ElseIf(...)` のカッコ内には，OKE ソフトの条件判定チップに相当する条件判定命令を記述します．

条件判定命令は大きく分けて 3種類あります．

- 単純条件判定
	- `isTargetStopped` のように，単純にその真偽を判定するチップは，その命令のみを記述します．
	- [命令一覧](#命令一覧)では[条件判定]と注釈してあります
- 数値大小比較
	- 武装残弾数判定 `numAmmo` 等，数値の大小関係を比較する命令は `>=` `<=` `>` `<` 比較演算子を用いて，より直感的に記述することができます．
	- [命令一覧](#命令一覧)では[数値大小判定]と注釈してあります
	- 補足: OKE ソフト仕様では `>` `<` 演算子はありませんが，例えば `比較命令 > 値` は `!(比較命令 <= 値)` に変換されます
- 数値同値比較
	- 被ロック数判定 `numLocked` 等，数値と等しいか判定する命令は `==` `!=` 比較演算子を用いて，より直感的に記述することができます．
	- [命令一覧](#命令一覧)では[数値同値判定]と注釈してあります
	- 補足: OKE ソフト仕様では `!=` 演算子はありませんが，`比較命令 != 値` は `!(比較命令 == 値)` に変換されます

### 論理演算子

条件判定命令には `!` `&&` `||` の論理演算子を使用することができます．

```
// 正面 50m 以内の地上敵に格闘攻撃をする
If(isTargetPosition.span(90).dist(50) && targetBodyCode != BC_FLIGHT)
	fight;
End

If(!isTargetStopped)
	// ターゲットが静止中ではないときの処理
	
End
```

## ループの記述

繰り返し処理を行うためのループ構文は `While` `End` `Break` を使用して記述します．以下に記述例を示します．
```
// ミサイルが 100m 以内にある限りループし続ける
While(numProjectile.dist(100).type(P_MISSILE) >= 1)
	
	// ただし 2発以上のときは妨害を発動してループを抜ける
	If(numProjectile.dist(100).type(P_MISSILE) >= 2)
		option(1); // ミサイル妨害
		Break;
	End
	
	// ミサイル 1発は機動で避ける，等
	...
End
```

ループを脱出するためには `Break` を使用してください．

## カウンタ入力

カウンタ代入する命令 `numAllEnemy` 等は，以下のように記述します．

```
A = numAllEnemy;
```

カウンタに代入するための命令は[命令一覧](#命令一覧)では[カウンタ入力]と注釈してあります

## CPU サイズ変更

標準では MAIN:15x15 + SUB1:7x7 + SUB2:7x7 の一番大きな CPU 向けにコンパイルされます．
4x4 等，他の CPU 向けにサイズを変更する場合は，`setCpuSize()` 命令を `chip_main()` の先頭に記述します．
```
void chip_main(){
	setCpuSize(4);
	...
}
```

# 命令一覧

以下に使用できる命令一覧を示します．

`<機体からの距離>` などの表記は，その表記が示す値を記述します．

[数値大小判定]命令に記載の `<数値>` は命令引数には指定せず，比較演算子 `>=` 等の右辺にその数値を記述します．

## 何もしない命令

`nop` : 何もしない (ノップ)

`sleep(<待ちカウント>)` : 思考停止

`wait` : 動作完了待ち

`isTargetPosition` : ターゲットが指定範囲であれば分岐

`targetBodyCode` : 機体コードが指定値と等しければ分岐

機体コードは[定義済み定数](#機体コード)を指定します．

## ターゲットロック警戒判断 [数値同値判定]

`numLocked` : 自機をターゲットロックしている機数を判定

## ターゲット武器判断 [数値同値判定]

`targetWeaponId(<武装番号>)` : ターゲットの武装が指定種別と等しいか判断

武装種別は[定義済み定数](#武装種別)を指定します．

## ターゲットまでの地表判断 [条件判定]

`isLineBlocked` : ターゲットまでの射線が遮られていれば分岐

## カウンタの値を判断

`<カウンタ1> >= <数値>` `<カウンタ1> >= <カウンタ2>` : <カウンタ1>が<数値/カウンタ2>以上なら分岐

`<カウンタ1> <= <数値>` `<カウンタ1> <= <カウンタ2>` : <カウンタ1>が<数値/カウンタ2>以下なら分岐

`<カウンタ1> == <数値>` `<カウンタ1> == <カウンタ2>` : <カウンタ1>が<数値/カウンタ2>であるならば分岐

`>` `<` `!=` も使用できます．

## 射撃攻撃

`fire(<武装番号>, <発射弾数>)` : <武装番号>を<発射弾数>発発射

### オプション

エリア指定 : [エリア指定オプション](#エリア指定オプション) を参照してください

`.target` : エリア指定ではなく，ロック中のターゲットに射撃します

`.wide` : 射撃モードを乱射 にします (移動射撃，ジャンプ射撃時は無視されます)

`.snipe` : 射撃モードを 狙撃 にします (移動射撃，ジャンプ射撃時は無視されます)

`.jumpForward` 
`.jumpBackward` 
`.jumpLeft` 
`.jumpRight` : 指定方向にジャンプしながら射撃します．自動的にターゲット射撃になります．

`.wait` : 実行モードを 停止 にします

## エリア指定オプション

エリア指定を行う命令では，次に示す共通のエリア指定オプションが使用できます．

極座標指定のオプションを 1つ以上指定すると極座標指定になり，
直交座標指定のオプションを 1つ以上指定すると直交座標指定になります．
どちらも指定されなかった場合は，極座標のデフォルト値が使用されます．

### 極座標指定

`.dist(<数値>)` : 距離，デフォルト値 = 800

`.dir(<数値>)` : 方位，デフォルト値 = 0

`.span(<数値>)` : 幅，デフォルト値 = 360


### 直交座標

`.w(<数値>)` : 横幅，デフォルト値 = 800

`.h(<数値>)` : 縦幅，デフォルト値 = 800

`.x(<数値>)` : x 位置，デフォルト値 = 0

`.y(<数値>)` : y 位置，デフォルト値 = 0
