# OKE Chip Compiler for ZEUS Carnage Heart Second

## 概要

OKE Chip Compiler (okecc) は，ゼウス カルネージハートセカンドのソフトを C 言語に似た[プログラミング言語](https://github.com/yoshinrt/okecc/blob/main/sample.cpp)で記述し，チップ配置を自動で行うソフトウェアです．
チップ配置ではなくプログラミング言語で OKE ソフトを設計することは，以下のような利点があります．

- 「チップを並べる」行為から開放されます
	- チップ移動や矢印修正，チップ整形，NOP 圧縮の手間がありません
- 言語 (テキスト) で書かれたソフトは，流用が容易です
- サブルーチン数に制限がありません (疑似サブルーチン)
	- 疑似サブルーチンはインライン展開されます
	- 実際の SUB1/2 に割り当てることも容易です

## 使用方法

1. 本ソフトの動作には Microsoft Visual Studio が必要ですのでインストールしてください．Microsoft Visual Studio Community 2022 で動作確認しています．
1. 本 git リポジトリをクローンします．
1. powershell プロンプトを立ち上げ，git リポジトリをクローンしたディレクトリに移動します．
1. 以下のコマンドを実行すると，サンプルの OKE ソフトがコンパイルされます．
	```
	./okecc.ps1 sample
	```

	「このシステムではスクリプトの実行が無効になっているため、ファイルを読み込むことができません。」というエラーが表示された場合は，PowerShell スクリプトの実行を許可する必要があります．[ここ](https://www.google.com/search?q=powershell+%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%97%E3%83%88%E3%81%AE%E5%AE%9F%E8%A1%8C%E3%82%92%E8%A8%B1%E5%8F%AF%E3%81%99%E3%82%8B%E3%81%AB%E3%81%AF)を参考に設定してください．

	別の OKE ソフトをコンパイルしたい場合は，例えば `hoge.cpp` に保存し， `./okecc.ps1 hoge` を実行することでコンパイルします．
1. コンパイル・チップ配置に成功すると，以下のファイルが生成されます．

	`sample.svg` : チップ配置結果をグラフィカルに表示するためファイル．Chrome 等の Web ブラウザで閲覧できます．

	`memcard.mcr` : DuckStation 等のメモリカードファイル．このファイルが存在し，Zeus / ZeusII のシナリオセーブデータが存在する場合，その OKE カード 1番目のソフトに結果を書き込みます．

### 実行時間について

チップ配置は力任せで探索しているため，実行時間はチップ数と PC の CPU パワーと運に依存します．

参考までに，Ryzen5 5600G での実行時間は以下のとおりです．

- 45チップ (sample.cpp): 1秒以下
- 176チップ (test_176chip.cpp): およそ 30-180秒

### 交差矢印について

本コンパイラが生成するチップ配置は，チップ配置成功率を上げるため，交差矢印を禁止していません．

## OKE ソフトの記述方法

### メインルーチンの記述

OKE ソフトは C 言語に似た言語で記述します．以下に，最小限の OKE ソフト記述を示します．

```
#include <okecc.h>

void chip_main(){
	// ここに命令を記述
}
```

`chip_main()` が，メインの 15x15 のエリアを意味し，ここに配置するプログラムを記述します．`chip_main()` は必ず必要です．

命令は，OKE ソフトチップに相当する命令を記述します．使用できる命令は[命令一覧](#命令一覧)を参照してください．

### exit() 命令

`exit()` 命令は，チップ配置エリアの外枠に向かうチップが配置されます．MAIN では MAIN 先頭に戻り，SUB1/SUB2 では呼び出し元に戻ります．

### 条件分岐の記述

条件分岐には，`if - elseif - else - endif` を使用して記述します．以下に記述例を示します．

```
if(ammo_num(1) >= 100)
	// 武装 1残弾が 100 発以上の時の処理
	...
elseif(is_target_stop())
	// 武装 1残弾が 100 未満 かつ ターゲットが静止中の処理
	...
else
	// 上記条件のいずれにも当てはまらないときの処理
	...
endif
```

`if(...)` および `elseif(...)` のカッコ内には，OKE ソフトの条件判定チップに相当する条件判定命令を記述します．

条件判定命令は大きく分けて 2種類あります．

- 単純条件判定
	- `is_target_stop()` のように，単純にその真偽を判定するチップは，その命令のみを記述します．
	- [命令一覧](#命令一覧)では[条件判定]と注釈してあります
- 数値比較
	- 武装残弾数判定 `ammo_num()` 等，数値の大小関係を比較する命令は `>=` `<=` といった比較演算子を用いて，より直感的に記述することができます．
	- [命令一覧](#命令一覧)では[数値判定]と注釈してあります

### ループの記述

繰り返し処理を行うためのループ構文は `loop` `endloop` `break` を使用して記述します．以下に記述例を示します．
```
loop
	...
	if(...)
		break;
	endif
	...
endloop
```

`loop - endloop` は無限ループなので，ループを脱出するために `break` を使用してください．

#### 論理演算子

条件判定命令には `!` `&&` `||` の論理演算子を使用することができます．

```
// 正面 30m 以内の地上敵に格闘攻撃をする
if(target_z() <= 6 && is_target_direction(0, 160) && target_distance() <= 30)
	strike();
endif

if(!is_target_stop())
	// ターゲットが静止中ではないときの処理
	
endif
```

### カウンタ入力

武装残弾数をカウンタ代入する命令 `ammo_num()` 等は，以下のように記述します．

```
A = ammo_num(1);
```

カウンタに代入するための命令は[命令一覧](#命令一覧)では[カウンタ入力]と注釈してあります

`ammo_num()` 等は，残弾数の判定と，残弾数のカウンタへの取得，と同じ命令名であることに注意してください．`ammo_num()` 自体は武装の残弾数を意味する命令で，`>=` 等によって比較するか，カウンタに代入するか，によって実際に配置されるチップが切り替えられます．

### サプルーチンの記述

以下に，サブルーチンの記述例を示します．


```
#include <okecc.h>

void sub_routine(){
	start_sub(1);
	
	// ここに SUB1 ソフトを記述
}

void chip_main(){
	...
	sub_routine();
	...
}
```

サプルーチンとして記述する関数 (ここでは `sub_routine()`) を記述し，その先頭で `start_sub(n)` を記述します．`n` には 1 または 2 を指定し，それぞれ SUB1, SUB2 に配置されます．また関数名は自由に定義できます．

`start_sub(n)` を記述しなかった関数は疑似サブルーチンとなり，この例では `chip_main()` の呼び出し位置に `sub_routine()` のチップがインライン展開されます．

## 命令一覧

以下に使用できる命令一覧を示します．

`<機体からの距離>` などの表記は，その表記が示す値を記述します．

[数値判定]命令に記載の `<数値>` は命令引数には指定せず，比較演算子 `>=` 等の右辺にその数値を記述します．

### 何もしない命令

`nop()` : 何もしない (0カウント間)

`wait(<待ちカウント>)` : 無いもしない(<待ちカウント>カウント間)

`wait_ae()` : 無いもしない(アクション終了まで)

### 移動動作命

`move_forward()` : 前進する

`move_badkward()` : 後退する

`move_right()` : 右移動する

`move_left()` : 左移動する

`stop()` : 静止する

### 回転動作

`turn_rignt()` : 右回りに旋回する

`turn_left()` : 左回りに旋回する

### ジャンプ

`jump_forward()` : 前へジャンプする

`jump_badkward()` : 後ろへジャンプする

`jump_right()` : 右へジャンプする

`jump_left()` : 左へジャンプする

### アクション

`strike()` : 格闘攻撃をする

`crouch()` : その場に伏せる

`guard()` : ガード姿勢を取る

`action(<番号>)` : 特殊アクション<番号>を実行

### 飛行高度指示

`move_low()` : 飛行高度を低空にする

`move_mid()` : 飛行高度を中空にする

`move_high()` : 飛行高度を上空にする

### 残弾数とオプション残弾 判定 [数値判定]

`ammo_num(<武装番号>)` : <武装番号>版武装が<数値>発以上/以下で分岐

`option_num(<オプション番号>)` : <オプション番号>番オプション残量が<数値>以上/以下ならば分岐

### 近くの敵に攻撃指示

### オプションの動作を指示

### 近くの機体を判断 [数値判定]

`enemy_num(<機体からの方向>, <指定方向の幅>, <機体からの距離>, <機体の種別>)` : 指定範囲<機体からの距離>M以内の敵<機体の種別>が<数値>機以上/以下で分岐

`friendly_num(<機体からの方向>, <指定方向の幅>, <機体からの距離>, <機体の種別>)` : 指定範囲<機体からの距離>M以内の味方<機体の種別>が<数値>機以上/以下で分岐

`is_barrier_under(<機体からの方向>, <指定方向の幅>, <機体からの距離>, <高さを指定>)` : 指定範囲<機体からの距離>M以内に<高さを指定>以下の障害物で分岐

### 近くの危険物を判断 [数値判定]

### 演算を実行

`<カウンタ1> += <数値>` `<カウンタ1> += <カウンタ2>` : <カウンタ1>に<数値/カウンタ2>を足す

`<カウンタ1> -= <数値>` `<カウンタ1> -= <カウンタ2>` : <カウンタ1>から<数値/カウンタ2>を引く

`<カウンタ1> *= <数値>` `<カウンタ1> *= <カウンタ2>` : <カウンタ1>に<数値/カウンタ2>を掛ける

`<カウンタ1> /= <数値>` `<カウンタ1> /= <カウンタ2>` : <カウンタ1>を<数値/カウンタ2>で割る

`<カウンタ1> %= <数値>` `<カウンタ1> %= <カウンタ2>` : <カウンタ1>を<数値/カウンタ2>で割った余りを入力

`<カウンタ1> = <数値>` `<カウンタ1> = <カウンタ2>` : <カウンタ1>に<数値/カウンタ2>を入力

### カウンタを指定範囲の値に変更

### カウンタの値を判断

`<カウンタ1> >= <数値>` `<カウンタ1> >= <カウンタ2>` : <カウンタ1>が<数値/カウンタ2>以上なら分岐

`<カウンタ1> <= <数値>` `<カウンタ1> <= <カウンタ2>` : <カウンタ1>が<数値/カウンタ2>以下なら分岐

`<カウンタ1> == <数値>` `<カウンタ1> == <カウンタ2>` : <カウンタ1>が<数値/カウンタ2>であるならば分岐

`<カウンタ1> != <数値>` `<カウンタ1> != <カウンタ2>` : <カウンタ1>が<数値/カウンタ2>でないならば分岐
